## 1. 개요

우량주 스크리닝은 **장기(6개월~수년) 관점**으로 사업 품질이 우수한 종목을 펀더멘털 기반으로 발굴한다.
AI 추천 파이프라인(tech/ML/뉴스 단기 신호)·가치주 스크리너(중기 저평가 발굴)와는 독립적으로 동작하는 별도 스크리너다.

> **투자 시계:** 6개월~수년. 기술적 지표(단기 5-60일), ML 모델(10거래일), 가치주 스크리너(3-6개월)와는 목적이 다르다.

주요 담당 모듈:
- `src/koreanstocks/core/engine/quality_screener.py` → `QualityScreener` (필터 + 점수)
- `src/koreanstocks/core/data/provider.py` → `get_value_candidates()` (후보군 수집)
- `src/koreanstocks/core/data/fundamental_provider.py` → `FundamentalProvider` (DART 수집)
- `src/koreanstocks/api/routers/quality.py` → `GET /api/quality_stocks`

```
후보군 수집 (시가총액 상위 N종목, ROE 사전 필터, PER 상한 없음)
  → 펀더멘털 병렬 수집 (DART API + SQLite 당일 캐시)
  → 필터 체인 (6단계 하드 필터)
  → ROE 2개년 평균 계산 (지속성 반영)
  → quality_score 산출 (0~100)
  → quality_score 내림차순 정렬
  → 상위 limit개 반환
```

### 가치주 스크리너와의 핵심 차이

| 항목 | 우량주 스크리너 | 가치주 스크리너 |
|------|--------------|--------------|
| **투자 시계** | 6개월~수년 (장기) | 3~6개월 (중기) |
| **핵심 기준** | 사업 품질 (고ROE·고마진·안정 성장) | 가격 (저평가 — 낮은 PER·PBR) |
| **PER 상한** | 없음 (비싸도 좋은 기업 포함) | 25.0 (고평가 제거) |
| **ROE 하한** | 12% (자본 효율 엄격 기준) | 8% (최저 기준) |
| **부채비율 상한** | 100% (건전 재무) | 150% (완화 기준) |
| **영업이익률** | 10% 이상 필수 | 제한 없음 |
| **정렬 기준** | quality_score (ROE·마진·YoY·부채·배당) | F-Score 우선, 동점 시 value_score |
| **대시보드 탭** | 우량주 추천 (5번째) | 가치주 추천 (4번째) |
| **CLI** | `koreanstocks quality` | `koreanstocks value` |

## 2. 후보군 수집

담당: `DataProvider.get_value_candidates(limit, market, per_max=9999, roe_min)`

우량주 스크리너는 **PER 제한 없이** ROE 사전 필터만 적용해 후보군을 수집한다.

### 2-1. 데이터 소스

네이버 금융 **시가총액 순위 페이지**(`finance.naver.com/sise/sise_market_sum.naver`)를 사용한다.
ROE 컬럼을 파싱해 `roe_min - 5` 관용 하한으로 사전 필터를 적용한다.

```
URL: https://finance.naver.com/sise/sise_market_sum.naver
     ?sosok={시장코드}&page={페이지번호}
  sosok: 0 = KOSPI, 1 = KOSDAQ
```

### 2-2. 가치주 스크리너와의 차이

| 항목 | 우량주 | 가치주 |
|------|--------|--------|
| `per_max` 전달값 | `9999` (제한 없음) | 실제 `per_max` 값 |
| `roe_min` 전달값 | `max(0.0, roe_min - 5)` | `roe_min - 5` |

가치주 스크리너에서 PER로 걸러지는 고성장·고PER 종목(바이오, IT플랫폼 등)이
우량주 후보군에는 포함된다.

### 2-3. 시장 필터

| `market` 값 | 수집 대상 |
|------------|----------|
| `ALL` | KOSPI + KOSDAQ 병합 (시가총액 합산 정렬) |
| `KOSPI` | KOSPI만 |
| `KOSDAQ` | KOSDAQ만 |

## 3. 펀더멘털 수집

담당: `FundamentalProvider.get_fundamentals_batch(codes, max_workers=15)`

가치주 스크리너와 동일한 인프라를 공유한다. 상세 내용은 `docs/VALUE_SCREENING.md § 3` 참조.

### 3-1. 우량주 전용 추가 수집 항목

우량주 스크리너는 **ROE 2개년 평균** 계산을 위해 `roe_prev`(전전년도 ROE)도 활용한다.
`FundamentalProvider`가 반환하는 `f` 딕셔너리 주요 키:

| 키 | 설명 |
|----|------|
| `roe` | 당해 연도 ROE (%) |
| `roe_prev` | 전년도 ROE (%) — 있으면 2개년 평균 적용 |
| `op_margin` | 영업이익률 (%) = 영업이익 / 매출액 × 100 |
| `op_income_yoy` | 영업이익 전년 대비 성장률 (%) |
| `debt_ratio` | 부채비율 (%) |
| `pbr` | 주가순자산비율 |
| `dividend_yield` | 배당수익률 (%) |
| `op_income_positive` | 영업이익 흑자 여부 (bool) |

### 3-2. DART 폴백 순서

```
1순위: (year-1, 11011) — 전년도 사업보고서
2순위: (year-1, 11012) — 전년도 반기보고서
3순위: (year-2, 11011) — 전전년도 사업보고서 (3월 이전 폴백)
```

### 3-3. SQLite 캐시

가치주 스크리너와 동일한 `fundamental_cache` 테이블을 공유한다. 당일 1회 DART 호출로 제한.

## 4. 필터 체인

6단계를 **순서대로** 적용하며, 하나라도 탈락하면 즉시 제외한다.

| 순서 | 필터 | 기본 임계값 | 가치주 대비 | 목적 |
|------|------|-----------|-----------|------|
| 1 | 영업이익 흑자 | 필수 | 동일 | 적자 기업 완전 배제 |
| 2 | **영업이익률** | ≥ 10.0% | **신규** — 가치주에 없음 | 저마진 박리다매 기업 제거 |
| 3 | ROE | ≥ 12.0% | 8% → **12%** (강화) | 자본 효율성 엄격 기준 |
| 4 | 영업이익 YoY | ≥ -10.0% | 없음 → **신규** | 이익 급감 기업 제거 (소폭 역성장 허용) |
| 5 | 부채비율 | ≤ 100% | 150% → **100%** (강화) | 건전 재무구조 기준 |
| 6 | PBR | ≤ 6.0 | 3.0 → **6.0** (완화) | 고성장 우량주 포함 허용 |

**데이터 없음 처리:**
- ROE와 영업이익률 모두 없으면 → 판단 불가 → 제외
- 영업이익률 없으면 → **필터 2 탈락** (핵심 기준 미확인)
- 영업이익 YoY 없으면 → 통과 허용 (신규 상장 등)
- 기타 항목 없으면 → 해당 필터 건너뜀

> **PER 상한 없음**: 가치주 스크리너는 PER ≤ 25로 고평가 주식을 제거하지만,
> 우량주 스크리너는 PER 제한 없이 비싸도 좋은 기업을 포함한다.

## 5. quality_score

담당: `quality_screener.quality_score(f)` → `float` (0~100)

5개 요소의 **가용 데이터 기반 정규화 점수**를 합산한다.
배당수익률은 데이터 없음을 0으로 처리하여 전 종목을 100pt 기준으로 비교한다.

### 5-1. 구성 요소

| 요소 | 배점 | 만점 조건 | 최저 조건 |
|------|------|---------|---------|
| **ROE** | 30pt | ROE ≥ 20% | ROE ≤ 0% → 0pt |
| **영업이익률** | 25pt | 영업이익률 ≥ 20% | 영업이익률 ≤ 0% → 0pt |
| **영업이익 YoY** | 20pt | 영이YoY ≥ 30% | 영이YoY ≤ 0% → 0pt |
| **부채비율** | 15pt | 부채비율 = 0% | 부채비율 ≥ 100% → 0pt |
| **배당수익률** | 10pt | 배당수익률 ≥ 3% | 무배당 → 0pt |

### 5-2. 계산 공식

```python
# ROE (30pt 만점)
roe_score = min(30.0, max(0.0, roe / 20 * 30))

# 영업이익률 (25pt 만점)
margin_score = min(25.0, max(0.0, op_margin / 20 * 25))

# 영업이익 YoY (20pt 만점 — 30% 이상에서 만점)
yoy_score = min(20.0, max(0.0, yoy / 30 * 20))

# 부채비율 (15pt 만점 — 100% 이상에서 0pt)
debt_score = max(0.0, 15 * (1 - min(debt, 100) / 100))

# 배당수익률 (10pt 만점 — None은 0으로 처리)
div = dividend_yield or 0.0
div_score = min(10.0, max(0.0, div / 3 * 10))

# 정규화
earned   = sum([roe_score, margin_score, yoy_score, debt_score, div_score])
possible = sum([30, 25, 20, 15, 10])   # 데이터 없는 항목은 양쪽 모두 제외
quality_score = round(earned / possible * 100, 1)
```

### 5-3. 계산 예시

ROE 18%, 영업이익률 15%, 영이YoY 25%, 부채비율 35%, 배당 1.5%인 종목:

| 요소 | 계산 | 득점 |
|------|------|------|
| ROE | 18/20 × 30 | 27.0pt |
| 영업이익률 | 15/20 × 25 | 18.75pt |
| 영이YoY | 25/30 × 20 | 16.67pt |
| 부채비율 | (1 - 35/100) × 15 | 9.75pt |
| 배당 | 1.5/3 × 10 | 5.0pt |
| **합계** | **77.2 / 100** | **77.2점** |

### 5-4. value_score와의 비교

| 항목 | quality_score | value_score |
|------|-------------|------------|
| PER 반영 | 없음 | 25pt (업종 중앙값 대비) |
| PBR 반영 | 없음 | 15pt |
| ROE | 30pt | 20pt |
| 영업이익률 | 25pt | 없음 |
| 영업이익 YoY | 20pt | 30pt |
| 부채비율 | 15pt | 15pt |
| 배당 | 10pt | 10pt (데이터 있을 때) |

## 6. ROE 2개년 평균

우량주 스크리너는 ROE 지속성을 반영하기 위해 **2개년 평균 ROE**를 사용한다.

```python
roe_cur  = f.get("roe")       # 당해 연도 ROE
roe_prev = f.get("roe_prev")  # 전년도 ROE

if roe_cur is not None and roe_prev is not None:
    roe = round((roe_cur + roe_prev) / 2, 1)   # 2개년 평균
else:
    roe = roe_cur                                # 단일 연도 사용
```

이 평균 ROE는 필터 체인(3단계)과 quality_score 계산 모두에 적용된다.

**효과:**
- 일회성 이익으로 ROE가 급등한 종목 방어 (예: 자산 매각)
- 구조적 고수익 기업과 일시적 고수익 기업 구분

## 7. 인메모리 캐시

담당: `QualityScreener._cache` (프로세스 내)

### 캐시 키

```python
cache_key = (market, roe_min, op_margin_min, yoy_min,
             debt_max, pbr_max, candidate_limit, limit)
```

필터 파라미터가 완전히 동일한 요청에 대해서만 캐시를 적용한다.

### 유효 기간 및 무효화

```python
today = date.today()
if self._cache_date != today:
    self._cache.clear()       # 날짜 변경 시 전체 초기화
    self._cache_date = today
```

| 상황 | 소요 시간 |
|------|---------|
| 최초 스크리닝 (100종목 탐색) | 1~2분 |
| 최초 스크리닝 (300종목 탐색) | 2~4분 |
| 동일 조건 재요청 (캐시 히트) | 0.001초 미만 |
| 날짜 변경 후 첫 요청 | 위와 동일 (재수집) |

> 캐시는 프로세스 내 메모리에 한정된다. 서버 재시작 시 초기화된다.

## 8. API 레퍼런스

### GET /api/quality_stocks

우량주 스크리닝 실행 및 결과 반환.

**쿼리 파라미터:**

| 파라미터 | 기본값 | 설명 |
|---------|-------|------|
| `market` | `ALL` | 시장 필터: `ALL` \| `KOSPI` \| `KOSDAQ` |
| `roe_min` | `12.0` | ROE 하한 (%) |
| `op_margin_min` | `10.0` | 영업이익률 하한 (%) |
| `yoy_min` | `0.0` | 영업이익 YoY 하한 (%) |
| `debt_max` | `100.0` | 부채비율 상한 (%) |
| `pbr_max` | `6.0` | PBR 상한 |
| `candidate_limit` | `200` | 탐색 범위 — 시가총액 상위 몇 종목을 후보로 볼지 (100~500) |

**응답 스키마:**

```json
{
  "stocks": [
    {
      "code": "005930",
      "name": "삼성전자",
      "market": "KOSPI",
      "sector": "반도체",
      "roe": 16.8,
      "op_margin": 14.3,
      "op_income_yoy": 22.1,
      "debt_ratio": 42.5,
      "pbr": 1.6,
      "dividend_yield": 2.4,
      "quality_score": 74.2
    }
  ],
  "total": 18
}
```

> `fundamentals` 상세 데이터는 응답에서 제거된다 (내부 계산용).
> PER은 우량주 스크리너의 필터 기준이 아니므로 응답에 포함되지 않는다.

### GET /api/quality_stocks/filters

```json

  "op_margin_min": 10.0,
  "yoy_min": -10.0,
  "debt_max": 100.0,
  "pbr_max": 6.0
```

| ROE | 자기자본이익률 — 2개년 평균 (%) |
| 영업이익률 | 영업이익 / 매출액 × 100 (%) |
| 영이YoY | 영업이익 전년 대비 성장률 (%) |
| 부채비율 | 부채총계 / 자본총계 × 100 (%) |
| PBR | 주가순자산비율 |
| 배당 | 배당수익률 (%) |
| 우량점수 | quality_score 0~100 (색상 배지) |

### 지표 설명 가이드 (초보자용)

탭 내 토글 펼침으로 각 지표의 직관적 설명을 제공한다:
- **우량주란?**: "사업 자체가 뛰어난 기업 — 주가가 비싸더라도 오래 들고 가면 그만큼 벌 수 있는 기업"
- **ROE**: "내가 투자한 돈 100원으로 1년에 얼마를 버는가" (12% = 은행 금리의 약 4배)
- **영업이익률**: "100원 팔아서 본업으로 얼마를 남기나" (10% 이상 = 사업 경쟁력 있음)
- **영이YoY**: "작년보다 본업 이익이 얼마나 늘었나" (0% 이상 = 성장 중)
- **부채비율**: "내 돈 100원 대비 빚이 얼마나 많나" (100% 이하 = 재무 건전)
- **우량점수**: ROE·영업이익률·영이YoY·부채비율·배당 5개 항목 종합 점수

## 11. 우량주 vs 가치주 — 개념 및 설계 철학

### 두 투자 철학의 기원

**가치투자(Benjamin Graham)**: "안전마진(Margin of Safety)" — 내재가치보다 싸게 사는 것이 핵심.
PER·PBR로 저평가된 종목을 발굴한다.

**퀄리티 투자(Warren Buffett, Charlie Munger)**: "공정한 가격에 훌륭한 기업을 사는 것이
싼 가격에 보통 기업을 사는 것보다 훨씬 낫다." — 높은 ROE, 지속 가능한 경쟁 우위(해자),
안정적 이익 성장을 우선시한다.

### 두 스크리너의 적합 상황

| 상황 | 권장 스크리너 |
|------|-------------|
| 시장이 전반적으로 하락 후 저점 탐색 | 가치주 스크리너 (저PER·저PBR) |
| 장기 코어 포트폴리오 구성 | **우량주 스크리너** (고ROE·고마진) |
| 경기 민감 업종 (소재·에너지) | 가치주 스크리너 |
| 소비재·IT·헬스케어 장기 보유 | **우량주 스크리너** |
| 배당 수익 + 주가 방어 | 두 스크리너 공통 통과 종목 |

### 세 스크리너의 투자 시계

```
단기 (1~2주)    ──→  AI 추천 탭    (tech/ML/뉴스 신호)
중기 (3~6개월)  ──→  가치주 탭     (저평가 해소 사이클)
장기 (6개월~)   ──→  우량주 탭     (사업 품질 장기 보유)
```

### 우량주의 위험 — "좋은 기업 = 좋은 주식"의 함정

- **밸류에이션 리스크**: 시장이 이미 우량성을 가격에 반영했다면 추가 상승 여지가 작다.
  PBR 상한(6.0)을 두는 이유 — 지나치게 비싼 종목은 제외.
- **고PER 정당화 필요**: 고ROE가 지속되어야 고PER이 유지된다. 사업 모델 해자 분석이 병행되어야 한다.
- **경기 민감 고ROE**: 반도체·정유 등 사이클 업종은 정점에서 고ROE를 보이다 급락할 수 있다.
  영업이익 YoY 필터(기본 0% 이상)가 1차 방어선 역할을 한다.

## 12. 설계 원칙 및 한계

### 고정값 (임의 수정 금지)

| 항목 | 값 | 이유 |
|------|----|------|
| 기본 필터 임계값 | `DEFAULT_QUALITY_FILTERS` 딕셔너리 | 백테스트 미검증 시 임의 변경 금지 |
| 정렬 기준 | `quality_score` 내림차순 | 수익성·마진·성장·재무 종합 |
| ROE 만점 기준 | 20% | 국내 상장사 상위권 ROE 수준 기준점 |
| 영업이익률 만점 기준 | 20% | 제조·IT 통합 기준 |
| 영업이익 YoY 만점 기준 | 30% | 고성장 변별력 확보 |

### 알려진 한계

| 항목 | 내용 | 영향도 |
|------|------|-------|
| **장기 특화** | 6개월~ 장기 관점. 단기(1~4주) 주가 변동은 tech/ML 모듈이 담당 | 높음 |
| 업종별 기준 미분화 | 영업이익률 10% 기준이 금융·유통 업종에는 불리 (업종 마진 차이 큼) | 높음 |
| ROE 2개년 평균 한계 | roe_prev 데이터 없으면 단일 연도만 사용 — 지속성 판단 부정확 | 중간 |
| DART 제출 시차 | 3월 기준 전년도 사업보고서 미제출 기업 많음 → 전전년도 폴백 | 중간 |
| 후보군 한계 | 시가총액 상위 N종목만 탐색 — 소형 우량주 누락 가능 | 중간 |
| 배당 품질 미검증 | 배당수익률만 반영, 배당 지속성·성장성 미확인 | 낮음 |
| 프로세스 재시작 시 캐시 소멸 | 인메모리 캐시 — 서버 재시작 후 첫 요청은 전체 재수집 필요 | 낮음 |

### 향후 개선 방향

| 우선순위 | 항목 | 기대 효과 |
|---------|------|---------|
| 1 | 업종별 ROE·영업이익률 중앙값 상대 평가 | IT·헬스케어 vs 유통·소재 공정 비교 |
| 2 | 3~5개년 ROE·영업이익률 추세 반영 | 일시적 실적 호전과 구조적 우량 기업 구분 |
| 3 | 이자보상배율(EBIT/이자비용) 추가 | 부채비율 보완 — 실질 채무 상환 능력 측정 |
| 4 | 결과 SQLite 저장 + 이력 조회 | 날짜별 우량주 스크리닝 히스토리 비교 |
| 5 | 자유현금흐름(FCF) 수익률 반영 | 이익의 현금 실현 여부 검증 |
